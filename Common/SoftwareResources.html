Interpreters

Python

The Python interpreter can be installed many different ways. Some common methods include a distribution's package manager (Linux), an installer (Windows and Mac), or as part of a package of bundled libraries and related software (multiple platforms). Some of these packages include the following:

  • Anaconda
  • Miniconda (Starting with a base system and adding stuff as necessary makes it a smaller installation)
  • Sage (Some useful instructions for integrating Jupyter notebooks are found here.)

Package Managers & Virtual Environments

Caution should be used when installing Python libraries within a Python installation that is used by (came with) the computer's operating system. Additionally, avoid using conflicting package managers to install Python components (within practicality). For example, pip3 is a Python3 package manager, and Apt is a Linux system package manager. The two do not know what each other has already installed or configured. Installing Python packages with both managers in the same location could lead to a corrupted installation, a particular mess if the installation is also used by the operating system. Virtual environments are often used to keep library installations separated. Anaconda/Miniconda use the conda package manager and the myenv option to create a virtual environment. (Please see the Miniconda page for more details.)
